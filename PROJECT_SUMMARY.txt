================================================================================
MODULE DESCRIPTIONS:
================================================================================

1. MAIN.PY - FastAPI Server Hub
   Purpose: Serves frontend, handles all API requests
   Key Routes:
   • GET /                    → Serves index.html
   • POST /simulate           → Runs simulation
   • GET /api-data            → Returns drivers/races
   • GET /get-laps/{race}     → Returns lap count
   • GET /lap-visualizer.*    → Serves visualizer files

2. PRECOMPUTE.PY - Database Generation
   Purpose: Pre-computes strategy data for all drivers/races
   Process:
   • Load driver performance data
   • Load track characteristics
   • Calculate optimal pit stop combinations
   • Generate tire degradation curves
   • Store in strategy_database.pkl

3. SIMULATE.PY - Core Simulation Engine
   Purpose: Runs race simulations with different strategies
   Main Function: run_simulation()
   Features:
   • Generates 3 pit stop strategies (1, 2, 3 stops)
   • Calculates lap times with tire degradation
   • Applies fuel/weight effects
   • Uses Genetic Algorithm for optimization
   • Runs Monte Carlo for uncertainty analysis
   • Generates realistic driver positioning

4. ADVANCED_ANALYSIS.PY - Analysis Calculations
   Purpose: Performs detailed race analysis
   Key Functions:
   • calculate_degradation_rate()   → Tire wear per lap
   • generate_detailed_breakdown()  → Lap-by-lap analysis
   • monte_carlo_analysis()         → 1000+ simulations
   • compare_strategies()           → Strategy comparison

5. INDEX.HTML - Main Frontend
   Purpose: Web interface for simulator
   Sections:
   • Input form (driver, race, pit loss)
   • Results summary (top 3 strategies)
   • Tire degradation display
   • Strategy charts (pie, line, composition)
   • Lap visualizer integration
   • Detailed analysis section

6. LAP-VISUALIZER.JS - Interactive Visualization
   Purpose: Lap-by-lap race visualization component
   Class: LapVisualizer
   Key Methods:
   • render()        → Create visualization UI
   • play/pause()    → Animation controls
   • setLap()        → Jump to specific lap
   • updateCharts()  → Real-time chart updates
   Features:
   ✓ Real-time lap animation
   ✓ Pit stop timeline
   ✓ Dynamic charts
   ✓ Strategy comparisons
   ✓ Responsive design

7. LAP-VISUALIZER.CSS - Styling
   Purpose: Professional styling for visualizer
   Key Classes:
   • .lv-container      → Main container
   • .lv-header         → Top section
   • .lv-controls-panel → Lap controls
   • .lv-main-content   → Content grid
   • .lv-chart-card     → Chart containers
   Design: Dark theme, gradients, animations

================================================================================
TROUBLESHOOTING:
================================================================================

Problem: Database not found
Solution: python precompute.py

Problem: Port 8000 in use
Solution: python main.py --port 8001

Problem: Visualization not showing
Solution: Check browser console (F12)
          Verify lap-visualizer files loading
          Try /visualizer-test.html

Problem: Slow simulation
Solution: Disable Monte Carlo (faster)
          Reduce GA population size
          Pre-generate more cache data
